### Key Spark Parameters Explained

<AccordionGroup>
  <Accordion title="Memory Configuration">
    **spark.driver.memory** / **spark.executor.memory**
    - Default: 1g (too low for production)
    - Recommended: 4-8g depending on data volume
    - Rule of thumb: 1-2GB per 10GB of data processed per batch
    
    **spark.memory.fraction**
    - Default: 0.6
    - Recommended: 0.8 for data-intensive workloads
    - Controls memory available for execution and storage
    
    **spark.memory.storageFraction**
    - Default: 0.5
    - Recommended: 0.3 for streaming (favor execution over caching)
  </Accordion>
  
  <Accordion title="Parallelism Configuration">
    **spark.sql.shuffle.partitions**
    - Default: 200
    - Recommended: 2-3x number of executor cores
    - For local mode: 50-100
    - For cluster mode: 200-400
    
    **spark.default.parallelism**
    - Default: Number of cores
    - Recommended: 2x number of available cores
    - Affects operations like groupBy, reduceByKey
    
    **spark.streaming.kafka.maxRatePerPartition**
    - Limits records consumed per partition per trigger
    - Prevents overwhelming the system
    - Start with 1000, increase gradually to 10000+
  </Accordion>
  
  <Accordion title="Delta Lake Optimization">
    **spark.databricks.delta.optimizeWrite.enabled**
    - Automatically optimizes file sizes during writes
    - Reduces small file problem
    - Recommended: true for production
    
    **spark.databricks.delta.autoCompact.enabled**
    - Automatically compacts small files
    - Improves read performance
    - Trade-off: Slightly slower writes
    
    **enableChangeDataFeed**
    - Required for CDC (Change Data Capture)
    - Already enabled in base configuration (stream_processor/main.py:49)
    - Essential for deduplication pipeline
  </Accordion>
</AccordionGroup>

#### Query Optimization

<CodeGroup>
```sql Optimized Queries
-- Use PREWHERE instead of WHERE for filtering
SELECT title, vote_average
FROM silver_layer.movies
PREWHERE release_date >= '2020-01-01'
WHERE popularity > 50
ORDER BY vote_average DESC
LIMIT 100;

-- Use appropriate aggregation functions
SELECT 
    toYYYYMM(release_date) as month,
    uniqExact(id) as unique_movies,  -- Use uniq() for approximate
    avg(vote_average) as avg_rating
FROM silver_layer.movies
GROUP BY month
ORDER BY month;

-- Materialize frequently used aggregations
CREATE MATERIALIZED VIEW silver_layer.movie_stats_by_year
ENGINE = SummingMergeTree()
ORDER BY release_year
AS SELECT
    toYear(release_date) as release_year,
    count() as movie_count,
    avg(vote_average) as avg_rating
FROM silver_layer.movies
GROUP BY release_year;
```

```sql Avoid These Patterns
-- DON'T: Use SELECT * on large tables
SELECT * FROM silver_layer.movies;  -- BAD

-- DO: Select only needed columns
SELECT id, title, release_date FROM silver_layer.movies;  -- GOOD

-- DON'T: Use WHERE on non-indexed columns for large scans
SELECT * FROM silver_layer.movies WHERE overview LIKE '%action%';  -- SLOW

-- DO: Use full-text search indexes or limit the scan
SELECT * FROM silver_layer.movies 
WHERE hasToken(overview, 'action') 
AND release_date >= '2020-01-01'  -- Reduce scan range
LIMIT 1000;
```
</CodeGroup>

#### Write Optimization

Optimize writes from Spark (batch_jobs/io/writers/clickhouse_jdbc_writer.py):

```python
class ClickHouseJdbcWriter:
    def write_table(self, df: DataFrame, table_name: str, mode: str = "append"):
        # Optimize DataFrame before writing
        df = df.repartition(10)  # Adjust based on data volume
        df = df.coalesce(5)  # Reduce to fewer partitions for writing
        
        df.write \
            .format("jdbc") \
            .option("driver", self.driver) \
            .option("url", self.url) \
            .option("user", self.username) \
            .option("password", self.password) \
            .option("isolationLevel", "NONE") \
            .option("dbtable", table_name) \
            .option("batchsize", "10000")  # Increase batch size
            .option("rewriteBatchedStatements", "true") \
            .mode(mode) \
            .save()
```

#### Connection Pooling

```python
# Implement connection pooling in Neo4j writer
from neo4j import GraphDatabase

class Neo4jWriter:
    def __init__(self, uri, username, password):
        self.driver = GraphDatabase.driver(
            uri, 
            auth=(username, password),
            max_connection_lifetime=3600,  # 1 hour
            max_connection_pool_size=50,
            connection_acquisition_timeout=60
        )
    
    def write_batch(self, records, batch_size=100):
        with self.driver.session() as session:
            for i in range(0, len(records), batch_size):
                batch = records[i:i+batch_size]
                session.write_transaction(self._create_nodes, batch)
```

### Delta Lake Optimization

#### Optimize and Z-Order

Regularly optimize Delta Lake tables to improve read performance:

```python
from delta.tables import DeltaTable

# Optimize with Z-ordering
delta_table = DeltaTable.forPath(spark, "s3a://deltalake/silver/valid/movies")

# Compact small files
delta_table.optimize().executeCompaction()

# Z-order by frequently filtered columns
delta_table.optimize().executeZOrderBy("release_date", "popularity")

# Vacuum old files (removes files older than retention period)
delta_table.vacuum(168)  # 7 days retention
```

<Warning>
  Run OPTIMIZE during low-traffic periods as it's resource-intensive.
</Warning>

#### Partition Strategy

```python
# Write with partitioning for better query performance
df.write \

# Query with partition pruning
filtered_df = spark.read \

    .load("s3a://deltalake/silver/valid/movies") \
    .where("data_type = 'movie' AND year = 2024 AND month = 1")
```

### Streaming Batch Size

Control micro-batch size in streaming queries:

```python

df.writeStream \
    .format("delta") \
    .option("checkpointLocation", checkpoint_path) \
    .trigger(processingTime="30 seconds") \  # Process every 30 seconds
    .start(table_path)

# Option 2: Limit records per trigger
df.writeStream \
    .format("delta") \
    .option("checkpointLocation", checkpoint_path) \
    .option("maxFilesPerTrigger", "10") \  # Process max 10 files per trigger
    .start(table_path)

# Option 3: Control Kafka consumption rate
.option("maxOffsetsPerTrigger", "10000")  # Max 10k records per trigger
```
