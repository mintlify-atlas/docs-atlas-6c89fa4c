## Overview

The ClickHouse to Neo4j pipeline builds a knowledge graph by reading data from ClickHouse and writing nodes and relationships to Neo4j. It creates nodes for movies, TV series, and persons, then establishes relationships (ACTED_IN, WORKS_ON) based on cast and crew information.

## Module Path

```python
from batch_jobs.pipelines.silver_gold.clickhouse_to_neo4j import write_clickhouse_to_neo4j
```

## Purpose

- **Node Creation**: Create Movie, TV_Series, and Person nodes in Neo4j
- **Relationship Management**: Establish ACTED_IN and WORKS_ON relationships
- **Incremental Updates**: Use diff columns to add/remove relationships efficiently
- **Graph Constraints**: Apply uniqueness constraints on node keys
- **Change Tracking**: Process only the latest batch version from ClickHouse

## Function Signatures

### `write_clickhouse_to_neo4j()`

Main entry point for the ClickHouse to Neo4j pipeline.

```python
def write_clickhouse_to_neo4j(transform_map=TRANSFORM_MAP):
    """
    Pipeline to write to Neo4j from ClickHouse
    """
```

<ParamField path="transform_map" type="dict" default="TRANSFORM_MAP">
  Configuration mapping for nodes and relationships
</ParamField>

**Returns**: None (writes to Neo4j)

**Process Flow**:
1. Load settings and initialize logging
2. Connect to Redis for version tracking
3. Create Spark session with ClickHouse and Neo4j connectors
4. Process nodes: Create Movie, TV_Series, and Person nodes
5. Process relationships: Create ACTED_IN and WORKS_ON relationships

### `process_nodes()`

Creates nodes in Neo4j from ClickHouse data.

```python
def process_nodes(
    transform_map: dict,
    settings: Settings,
    redis_client: RedisClient,
    clickhouse_reader: ClickHouseReader,
    neo4j_writer: Neo4jWriter
)
```

<ParamField path="transform_map" type="dict" required>
  Node configuration from `TRANSFORM_MAP["nodes"]`
</ParamField>

<ParamField path="settings" type="Settings" required>
  Configuration settings loaded from YAML
</ParamField>

<ParamField path="redis_client" type="RedisClient" required>
  Redis client for version tracking
</ParamField>

<ParamField path="clickhouse_reader" type="ClickHouseReader" required>
  Reader for ClickHouse tables
</ParamField>

<ParamField path="neo4j_writer" type="Neo4jWriter" required>
  Writer for Neo4j using Spark Neo4j connector
</ParamField>

### `process_relationships()`

Creates relationships in Neo4j by joining ClickHouse tables.

```python
def process_relationships(
    transform_map: dict,
    settings: Settings,
    redis_client: RedisClient,
    clickhouse_reader: ClickHouseReader,
    neo4j_writer: Neo4jWriter
)
```

**Process**:
1. Read main table (movie/tv_series) and related table (cast/crew)
2. Parse diff column to extract added/removed IDs
3. Join tables to get full relationship data
4. Write relationships to Neo4j with properties

### `join_to_get_relationship()`

Helper function to create relationship DataFrames.

```python
def join_to_get_relationship(
    left_df: DataFrame,
    right_df: DataFrame,
    key_cols: list[str],
    diff_col: str,
    action_col: str,
    id_col_in_diff: str,
    relationship_properties: list[str]
) -> DataFrame
```

<ParamField path="left_df" type="DataFrame" required>
  Main DataFrame with diff column (e.g., movie table)
</ParamField>

<ParamField path="right_df" type="DataFrame" required>
  Related DataFrame with IDs (e.g., movie_cast table)
</ParamField>

<ParamField path="key_cols" type="list[str]" required>
  Columns to join on (e.g., `["movie_id"]`)
</ParamField>

<ParamField path="diff_col" type="string" required>
  Column containing diff JSON (e.g., `"casts_diff"`)
</ParamField>

<ParamField path="action_col" type="string" required>
  Action to extract: `"added"` or `"removed"`
</ParamField>

<ParamField path="id_col_in_diff" type="string" required>
  ID field name in diff array (e.g., `"person_id"`)
</ParamField>

<ParamField path="relationship_properties" type="list[str]" required>
  Properties to include in the relationship
</ParamField>

## Diff Column Processing

The pipeline processes diff columns to efficiently update relationships:

**Example diff column**:
```json
{
  "added": [123, 456, 789],
  "removed": [111, 222]
}
```

**Processing logic**:
1. Parse JSON diff column using `from_json()`
2. Explode `added` array to get list of IDs to add
3. Explode `removed` array to get list of IDs to remove
4. Join with cast/crew table to get full relationship data
5. Write relationships to Neo4j (Neo4j connector handles merges)

## Cypher Query Examples

After running the pipeline, you can query the graph:

<CodeGroup>
```cypher Find Actors in a Movie
MATCH (p:Person)-[r:ACTED_IN]->(m:Movie {movie_id: 550})
RETURN p.name, r.character
ORDER BY r.cast_id
```

```cypher Find Director's Movies
MATCH (p:Person)-[r:WORKS_ON]->(m:Movie)
WHERE r.job = 'Director'
RETURN p.name, m.original_title, m.release_date
ORDER BY m.release_date DESC
```

```cypher Actor Collaboration Network
MATCH (p1:Person)-[:ACTED_IN]->(m:Movie)<-[:ACTED_IN]-(p2:Person)
WHERE p1.person_id < p2.person_id
RETURN p1.name, p2.name, COUNT(m) AS collaborations
ORDER BY collaborations DESC
LIMIT 10
```

```cypher TV Series Cast
MATCH (p:Person)-[r:ACTED_IN]->(t:TV_Series {tv_series_id: 1396})
RETURN p.name, r.character
ORDER BY p.name
```
</CodeGroup>

## Performance Considerations

- **Batch Version Filtering**: Only processes latest batch from ClickHouse
- **Repartitioning**: Uses `repartition_cols` to distribute write load
- **Constraints**: Creates uniqueness constraints on node keys for efficient merges
- **Parallel Processing**: Can process multiple data types and relationships in parallel
- **Spark-Neo4j Connector**: Uses optimized connector for bulk writes

## Error Handling

<Note>
  The Neo4j writer uses the Spark Neo4j connector's `MERGE` mode by default, which:
  - Creates nodes if they don't exist
  - Updates nodes if they already exist
  - Prevents duplicate nodes based on key constraints
</Note>

<Warning>
  Ensure Neo4j has sufficient memory for large batch writes. Consider adjusting:
  - `dbms.memory.heap.max_size`
  - `dbms.memory.pagecache.size`
</Warning>
