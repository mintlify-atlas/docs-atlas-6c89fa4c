## Overview

The Bronze to Silver pipeline deduplicates timestamped data and transforms raw JSON into structured schemas. It reads from the Bronze layer in Delta Lake (MinIO), applies timestamp-based deduplication, parses raw data, generates hash columns for change tracking, and writes to the Silver layer.

## Module Path

```python
from batch_jobs.pipelines.bronze_silver.minio_to_minio import run_dedup_timestamp
```

## Purpose

- **Deduplication**: Remove duplicate records based on timestamp within each batch
- **Schema Parsing**: Convert raw JSON data into structured Spark schemas
- **Hash Generation**: Create hash columns for tracking changes in arrays (casts, crews) and vector information
- **Incremental Processing**: Track batch versions in Redis to process only new data
- **Change Data Capture**: Use Delta Lake CDF (Change Data Feed) to efficiently read incremental changes

## Function Signature

### `run_dedup_timestamp()`

Main entry point for the Bronze to Silver pipeline.

```python
def run_dedup_timestamp():
    """
    Pipeline to dedup timestamp, from bronze to silver layer in Delta Lake Minio
    """
```

**Parameters**: None (reads from configuration)

**Returns**: None (writes to Delta Lake)

**Process Flow**:
1. Load settings from `settings.yaml`
2. Initialize Redis client for version tracking
3. Create Spark session with MinIO configuration
4. For each data type (movie, tv_series, person):
   - Retrieve last processed version from Redis
   - Read incremental changes using Delta Lake CDF
   - Apply timestamp-based deduplication
   - Parse raw JSON to structured schema
   - Generate hash columns for change tracking
   - Upsert to Silver layer Delta table
   - Update version in Redis

## Error Handling

<Warning>
  If Redis does not contain a version key for a data type, the pipeline will initialize it to 0 and process all data from the beginning.
</Warning>

```python
if not last_version:
    logger.info("No last version found for %s, setting to 0", version_key)
    redis_client.set(version_key, 0)
    last_version = 0
```

<Note>
  The pipeline uses Delta Lake's merge operation with conditions:
  - **Update**: When keys match AND source timestamp > target timestamp
  - **Insert**: When keys don't match (new records)
</Note>

## Performance Considerations

- **Incremental Processing**: Only reads new Delta Lake versions using CDF
- **Deduplication**: Window functions partition by key columns for efficient deduplication
- **Hash-based Change Detection**: Avoids full comparisons by using hash columns
- **Batch Version Tracking**: Redis stores last processed version to avoid reprocessing

## Dependencies

- **PySpark**: Data processing framework
- **Delta Lake**: ACID transactions and time travel
- **Redis**: Version tracking and coordination
- **MinIO**: S3-compatible object storage
