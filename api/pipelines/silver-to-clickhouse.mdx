## Overview

The Silver to ClickHouse pipeline reads deduplicated data from the Silver layer in Delta Lake and writes it to ClickHouse tables. It transforms nested Spark DataFrames into flat ClickHouse schemas, explodes array columns for cast and crew data, and preserves diff columns for downstream change tracking.

## Module Path

```python
from batch_jobs.pipelines.silver_silver.minio_to_clickhouse import write_minio_to_clickhouse
```

## Purpose

- **Schema Transformation**: Convert nested Spark schemas to flat ClickHouse tables
- **Array Explosion**: Create separate cast and crew tables from nested arrays
- **Batch Processing**: Load only the latest batch version from Silver layer
- **Table Initialization**: Automatically create ClickHouse tables if they don't exist
- **Change Tracking**: Preserve diff columns for Neo4j relationship processing

## Function Signatures

### `write_minio_to_clickhouse()`

Main entry point for the Silver to ClickHouse pipeline.

```python
def write_minio_to_clickhouse():
    """
    Pipeline to write to Clickhouse from Delta Lake Minio (After dedup timestamp)
    """
```

**Parameters**: None (reads from configuration)

**Returns**: None (writes to ClickHouse)

**Process Flow**:
1. Load settings and initialize logging
2. Initialize ClickHouse tables (create if not exist)
3. Create Spark session with MinIO and ClickHouse connectors
4. For each data type (movie, tv_series, person):
   - Read batch version from Redis
   - Filter Silver layer data by batch version
   - Apply transformation functions to prepare ClickHouse schemas
   - Write to ClickHouse tables using native protocol

### `process_table()`

Processes data for all data types and writes to ClickHouse.

```python
def process_table(
    settings: Settings,
    redis_client: RedisClient,
    delta_minio_reader: DeltaMinioReader,
    clickhouse_writer: ClickHouseNativeWriter,
    transform_map: dict = TRANSFORM_MAP
)
```

<ParamField path="settings" type="Settings" required>
  Configuration settings loaded from YAML
</ParamField>

<ParamField path="redis_client" type="RedisClient" required>
  Redis client for version tracking
</ParamField>

<ParamField path="delta_minio_reader" type="DeltaMinioReader" required>
  Reader for Delta Lake tables on MinIO
</ParamField>

<ParamField path="clickhouse_writer" type="ClickHouseNativeWriter" required>
  Writer for ClickHouse using native protocol
</ParamField>

<ParamField path="transform_map" type="dict" default="TRANSFORM_MAP">
  Mapping of data types to transformation functions
</ParamField>

## Transform Map Configuration

The `TRANSFORM_MAP` defines which ClickHouse tables to create for each data type:

```python
TRANSFORM_MAP = {
    "movie": [
        {"table_name": "movie", "transform_func": prepare_table_movie},
        {"table_name": "movie_cast", "transform_func": prepare_table_movie_cast},
        {"table_name": "movie_crew", "transform_func": prepare_table_movie_crew},
    ],
    "person": [
        {"table_name": "person", "transform_func": prepare_table_person}
    ],
    "tv_series": [
        {"table_name": "tv_series", "transform_func": prepare_table_tv_series},
        {"table_name": "tv_series_cast", "transform_func": prepare_table_tv_series_cast},
        {"table_name": "tv_series_crew", "transform_func": prepare_table_tv_series_crew},
    ]
}
```

## ClickHouse Table Schemas

### Movie Table
```sql
CREATE TABLE movie (
    movie_id Int64,
    original_title String,
    overview String,
    popularity Float64,
    release_date Date,
    vote_average Float64,
    vote_count Int64,
    genres String,  -- JSON
    belongs_to_collection String,  -- JSON
    production_countries String,  -- JSON
    vector_info_hash Int64,
    casts_total_hash Int64,
    crews_total_hash Int64,
    vector_info_hash_diff Bool,
    casts_diff String,  -- JSON with added/removed arrays
    crews_diff String,  -- JSON with added/removed arrays
    batch_version Int64
) ENGINE = MergeTree()
ORDER BY movie_id;
```

### Movie Cast Table
```sql
CREATE TABLE movie_cast (
    movie_id Int64,
    cast_id Int64,
    person_id Int64,
    character String,
    credit_id String,
    known_for_department String,
    batch_version Int64
) ENGINE = MergeTree()
ORDER BY (movie_id, cast_id);
```

### Person Table
```sql
CREATE TABLE person (
    person_id Int64,
    name String,
    gender Int32,
    also_known_as Array(String),
    biography String,
    birthday Date,
    deathday Nullable(Date),
    place_of_birth String,
    known_for_department String,
    batch_version Int64
) ENGINE = MergeTree()
ORDER BY person_id;
```

## Error Handling

<Warning>
  **Critical Error**: If the batch version is not found in Redis, the pipeline will abort.
  
  This is a safety measure to prevent data inconsistency. The Bronze to Silver pipeline must run successfully before this pipeline.
</Warning>

```python
if not last_version:
    logger.error(f"CRITICAL: Last version for {version_key} NOT found in Redis. Aborting pipeline.")
    sys.exit(1)
```

<Note>
  If a data type is not found in the transform map, the pipeline logs a warning and skips that data type.
</Note>

## Diff Column Format

The `casts_diff` and `crews_diff` columns contain JSON with this structure:

```json
{
  "added": [123, 456, 789],      // IDs added in this batch
  "removed": [111, 222]          // IDs removed in this batch
}
```

These diff columns are crucial for the [ClickHouse to Neo4j pipeline](/api/pipelines/clickhouse-to-neo4j) to efficiently update graph relationships.

## Performance Considerations

- **Native Protocol**: Uses ClickHouse native protocol (port 9000) for faster bulk inserts
- **Batch Version Filtering**: Only processes the latest batch to avoid full table scans
- **Parallel Writes**: Can write to multiple ClickHouse tables concurrently
- **Schema Optimization**: Flat schemas optimized for OLAP queries

## Dependencies

- **PySpark**: Data processing framework
- **Delta Lake**: Read Silver layer data
- **ClickHouse**: OLAP database for analytics
- **Redis**: Version tracking
- **MinIO**: S3-compatible object storage
