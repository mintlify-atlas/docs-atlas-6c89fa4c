## Overview

The ClickHouse to Pinecone pipeline generates vector embeddings from movie and TV series text data (overview and tagline) and indexes them in Pinecone. This enables semantic search capabilities across entertainment content using natural language queries.

## Module Path

```python
from batch_jobs.pipelines.silver_gold.clickhouse_to_pinecone import write_clickhouse_to_pinecone
```

## Purpose

- **Text Embedding**: Generate dense vector embeddings from text using transformer models
- **Semantic Search**: Enable natural language search across movies and TV series
- **Namespace Isolation**: Separate movie and TV series vectors into different namespaces
- **Incremental Updates**: Process only the latest batch version from ClickHouse
- **Batch Processing**: Efficiently encode large text datasets using GPU acceleration

## Function Signatures

### `write_clickhouse_to_pinecone()`

Main entry point for the ClickHouse to Pinecone pipeline.

```python
def write_clickhouse_to_pinecone(transform_map=TRANSFORM_MAP):
    """
    Pipeline to write to Pinecone from ClickHouse
    """
```

<ParamField path="transform_map" type="dict" default="TRANSFORM_MAP">
  Configuration mapping for vector generation
</ParamField>

**Returns**: None (writes to Pinecone)

**Process Flow**:
1. Load settings and initialize logging
2. Connect to Redis for version tracking
3. Create Spark session with ClickHouse connector
4. For each table (movie, tv_series):
   - Read latest batch from ClickHouse
   - Concatenate text columns (overview, tagline)
   - Generate vector embeddings using transformer model
   - Write to Pinecone with namespace and metadata

### `prepare_vector_schema()`

Transforms ClickHouse data into Pinecone vector format.

```python
def prepare_vector_schema(
    df: DataFrame,
    id_col: str,
    namespace: str,
    vector_prepare_cols: list[str],
    model_name: str = "sentence-transformers/all-MiniLM-L6-v2",
    vector_col_name: str = "document",
    metadata: str = "None"
) -> DataFrame
```

<ParamField path="df" type="DataFrame" required>
  Input DataFrame from ClickHouse
</ParamField>

<ParamField path="id_col" type="string" required>
  Column to use as Pinecone vector ID (e.g., `"movie_id"`)
</ParamField>

<ParamField path="namespace" type="string" required>
  Pinecone namespace for organizing vectors
</ParamField>

<ParamField path="vector_prepare_cols" type="list[str]" required>
  Text columns to concatenate for embedding (e.g., `["overview", "tagline"]`)
</ParamField>

<ParamField path="model_name" type="string" default="sentence-transformers/all-MiniLM-L6-v2">
  HuggingFace transformer model for embedding generation
  
  Recommended models:
  - `intfloat/e5-large-v2` (1024 dims, high quality)
  - `sentence-transformers/all-MiniLM-L6-v2` (384 dims, fast)
  - `BAAI/bge-large-en-v1.5` (1024 dims, state-of-the-art)
</ParamField>

<ParamField path="vector_col_name" type="string" default="document">
  Name for concatenated text column
</ParamField>

<ParamField path="metadata" type="string" default="None">
  JSON string with additional metadata to store with vectors
</ParamField>

**Returns**: DataFrame with Pinecone schema:
- `id` (String): Vector ID
- `namespace` (String): Pinecone namespace
- `values` (Array[Float]): Dense vector embedding
- `metadata` (String): JSON metadata
- `sparse_values` (Struct): Sparse vector (null for dense-only)

### `prepare_vector_col()`

Concatenates text columns for embedding.

```python
def prepare_vector_col(
    df: DataFrame,
    vector_prepare_cols: list[str],
    vector_col_name: str = "document"
) -> DataFrame
```

**Process**: Concatenates columns using ` | ` separator

**Example**:
```python
# Input columns:
# overview: "A computer hacker learns..."
# tagline: "Welcome to the Real World."

# Output document:
# "A computer hacker learns... | Welcome to the Real World."
```

### `get_embed_udf()`

Creates a Pandas UDF for batch embedding generation.

```python
def get_embed_udf(model_name: str = "sentence-transformers/all-MiniLM-L6-v2")
```

**Returns**: Pandas UDF that:
- Loads transformer model once per executor
- Processes texts in batches of 64
- Normalizes embeddings for cosine similarity
- Returns list of floats for each text

## Embedding Models

<Card title="E5 Large v2" icon="brain">

  **Performance**: High quality embeddings for semantic search
  
  **Usage**: Prefix queries with "query: " and passages with "passage: "

## Pinecone Index Configuration

Create a Pinecone index before running the pipeline:

```python
import pinecone

pinecone.init(
    api_key="your-api-key",
    environment="us-west1-gcp"
)

pinecone.create_index(
    name="entertainment-search",
    dimension=1024,  # Must match model dimensions
    metric="cosine",  # cosine, euclidean, or dotproduct
    pod_type="p1.x1"  # Performance tier
)
```

## Semantic Search Examples

After indexing, query the vectors using Pinecone:

<CodeGroup>
```python Python SDK
import pinecone
from sentence_transformers import SentenceTransformer

# Initialize
pinecone.init(api_key="...", environment="...")
index = pinecone.Index("entertainment-search")
model = SentenceTransformer("intfloat/e5-large-v2")

# Generate query vector
query = "movies about time travel and paradoxes"
query_vector = model.encode("query: " + query).tolist()

# Search movies
results = index.query(
    vector=query_vector,
    namespace="movie",
    top_k=10,
    include_metadata=True
)

for match in results['matches']:
    print(f"Movie ID: {match['id']}, Score: {match['score']}")
```

```python Hybrid Search (Text + Filters)
# Search with metadata filters
results = index.query(
    vector=query_vector,
    namespace="movie",
    top_k=10,
    filter={
        "batch_version": {"$gte": 40},
        "genre": "Science Fiction"
    }
)
```

```python Multi-namespace Search
# Search both movies and TV series
movie_results = index.query(
    vector=query_vector,
    namespace="movie",
    top_k=5
)

tv_results = index.query(
    vector=query_vector,
    namespace="tv_series",
    top_k=5
)

# Combine and re-rank
all_results = sorted(
    movie_results['matches'] + tv_results['matches'],
    key=lambda x: x['score'],
    reverse=True
)[:10]
```
</CodeGroup>

## Performance Considerations

<Card title="Optimization Tips" icon="gauge-high">
  **Batch Size**: UDF processes 64 texts per batch
  
  **GPU Acceleration**: Use GPU-enabled Spark executors for faster encoding
  
  **Model Caching**: Model loads once per executor, not per batch
  
  **Normalization**: Embeddings normalized for faster cosine similarity
  
  **Partitioning**: Repartition DataFrame before embedding for parallelism
</Card>

### Performance Metrics

| Model | Dimensions | Speed | GPU Memory |
|-------|-----------|-------|------------|
| all-MiniLM-L6-v2 | 384 | ~1000 texts/sec | 1 GB |
| e5-large-v2 | 1024 | ~400 texts/sec | 4 GB |
| bge-large-en-v1.5 | 1024 | ~350 texts/sec | 6 GB |

## Error Handling

<Warning>
  **Model Download**: The first run downloads the model (~1.5 GB for e5-large-v2). Ensure executors have internet access or pre-download models.
</Warning>

<Note>
  **Null Handling**: If `overview` or `tagline` is null, concatenation still works:
  ```python
  concat_ws(" | ", null, "Some text") → "Some text"
  ```
</Note>

## Advanced: Custom Metadata

You can enrich vectors with metadata for filtering:

```python
import json

custom_transform_map = {
    "movie": {
        "id_col": "movie_id",
        "vector_prepare_cols": ["overview", "tagline"],
        "vector_col_name": "document",
        "metadata": json.dumps({
            "type": "movie",
            "source": "tmdb",
            "indexed_at": "2024-01-15"
        })
    }
}
```

Then filter during search:
```python
results = index.query(
    vector=query_vector,
    filter={"type": "movie", "source": "tmdb"},
    top_k=10
)
```
