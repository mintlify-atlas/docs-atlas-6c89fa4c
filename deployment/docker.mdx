## Overview

Docker Compose provides a simple way to run the entire Entertainment Data Platform stack on your local machine. This is ideal for development, testing, and rapid prototyping.

## Prerequisites

<CardGroup cols={2}>
  <Card title="Docker" icon="docker">
    Docker Engine 20.10 or later
  </Card>
  <Card title="Docker Compose" icon="layer-group">
    Docker Compose v2.0 or later
  </Card>
</CardGroup>

## Services Included

The Docker Compose setup includes all the infrastructure services needed for the platform:

| Service | Purpose | Ports |
|---------|---------|-------|
| **Kafka (broker1)** | Message broker for event streaming | `9092` |
| **MinIO** | S3-compatible object storage for Delta Lake | `9000` (API), `9001` (Console) |
| **ClickHouse** | OLAP database for analytics | `8123` (HTTP), `9002` (Native) |
| **Tabix** | Web UI for ClickHouse | `8081` |
| **Redis** | In-memory cache and state management | `6379` |

## Quick Start

<Steps>
  <Step title="Navigate to Docker Directory">
    ```bash
    cd deployment/docker
    ```
  </Step>

  <Step title="Start All Services">
    ```bash
    docker compose up -d
    ```
    
    This will start all services in detached mode.
  </Step>

  <Step title="Verify Services are Running">
    ```bash
    docker compose ps
    ```
    
    All services should show status as "Up" or "healthy".
  </Step>
</Steps>

## Service Configuration

### Kafka Broker

Kafka runs in KRaft mode (without ZooKeeper) with the following configuration:

<CodeGroup>
```yaml docker-compose.yml
broker1:
  image: apache/kafka:latest
  hostname: broker1
  container_name: broker1
  ports:
    - 9092:9092
  environment:
    KAFKA_BROKER_ID: 1
    KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://broker1:29092,PLAINTEXT_HOST://localhost:9092
    KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
    KAFKA_LOG_RETENTION_MS: "9000000"
    KAFKA_LOG_RETENTION_BYTES: "2147483648"
    KAFKA_AUTO_CREATE_TOPICS_ENABLE: "true"
    KAFKA_NUM_PARTITIONS: 3
    CLUSTER_ID: MkU3OEVBNTcwNTJENDM2Qk
```
</CodeGroup>

**Key Configuration:**
- Auto-creates topics when first written to
- 3 partitions per topic for parallelism
- 2GB log retention per partition
- Data retention: ~2.5 hours (9,000,000 ms)

### MinIO Object Storage

<CodeGroup>
```yaml docker-compose.yml
minio:
  image: minio/minio
  container_name: minio
  ports:
    - 9000:9000  # S3 API
    - 9001:9001  # Web Console
  environment:
    MINIO_ROOT_USER: minio
    MINIO_ROOT_PASSWORD: minio123
  command: server --console-address ":9001" /data
```
</CodeGroup>

**Access Points:**
- **API Endpoint**: `http://localhost:9000`
- **Console**: `http://localhost:9001`
- **Credentials**: `minio` / `minio123`

### ClickHouse Database

<CodeGroup>
```yaml docker-compose.yml
clickhouse:
  image: clickhouse/clickhouse-server
  container_name: clickhouse
  ports:
    - 8123:8123  # HTTP interface
    - 9002:9000  # Native protocol
  environment:
    - CLICKHOUSE_DEFAULT_ACCESS_MANAGEMENT=1
```
</CodeGroup>

**Access:**
- **HTTP Interface**: `http://localhost:8123`
- **Tabix UI**: `http://localhost:8081`
- Default access management enabled

### Redis Cache

<CodeGroup>
```yaml docker-compose.yml
redis:
  image: redis:8.2.1-alpine
  ports:
    - "6379:6379"
```
</CodeGroup>

**Connection**: `redis://localhost:6379`

## Volume Persistence

<Warning>
  The default Docker Compose setup does not persist data across container restarts. All data is stored in container volumes that are destroyed when containers are removed.
</Warning>

To add persistent volumes, modify `docker-compose.yml`:

```yaml
services:
  minio:
    volumes:
      - ./data/minio:/data
  
  clickhouse:
    volumes:
      - ./data/clickhouse:/var/lib/clickhouse
  
  redis:
    volumes:
      - ./data/redis:/data
```

## Managing Services

### View Service Logs

<CodeGroup>
```bash All Services
docker compose logs -f
```

```bash Specific Service
docker compose logs -f broker1
```

```bash Last 100 Lines
docker compose logs --tail=100 minio
```
</CodeGroup>

### Stop Services

<CodeGroup>
```bash Stop (Keep Containers)
docker compose stop
```

```bash Stop and Remove
docker compose down
```

```bash Remove with Volumes
docker compose down -v
```
</CodeGroup>

### Restart Individual Service

```bash
docker compose restart broker1
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Kafka broker not starting">
    **Symptom**: Broker exits immediately after starting
    
    **Solution**:
    1. Check if port 9092 is already in use:
       ```bash
       lsof -i :9092
       ```
    2. Ensure the CLUSTER_ID is unique if running multiple instances
    3. Clear Kafka logs if data is corrupted:
       ```bash
       docker compose down -v
       docker compose up -d
       ```
  </Accordion>

  <Accordion title="Cannot connect to MinIO">
    **Symptom**: Connection refused to localhost:9000
    
    **Solution**:
    1. Verify MinIO is running:
       ```bash
       docker compose ps minio
       ```
    2. Check MinIO logs:
       ```bash
       docker compose logs minio
       ```
    3. Ensure no firewall is blocking ports 9000/9001
  </Accordion>

  <Accordion title="ClickHouse out of memory">
    **Symptom**: ClickHouse container keeps restarting
    
    **Solution**:
    1. Increase Docker memory limit (minimum 4GB recommended)
    2. Add memory limits to docker-compose.yml:
       ```yaml
       clickhouse:
         deploy:
           resources:
             limits:
               memory: 2G
       ```
  </Accordion>

  <Accordion title="Network connectivity issues between services">
    **Symptom**: Services can't reach each other
    
    **Solution**:
    1. All services are on the `test-network` bridge network
    2. Use service names (e.g., `broker1`, `minio`) not `localhost` for inter-service communication
    3. Recreate the network:
       ```bash
       docker compose down
       docker network prune
       docker compose up -d
       ```
  </Accordion>
</AccordionGroup>

## Health Checks

Monitor service health:

<CodeGroup>
```bash Kafka
docker exec broker1 sh -c "nc -z broker1 29092 && echo 'Kafka is healthy'"
```

```bash MinIO
curl -I http://localhost:9000/minio/health/live
```

```bash ClickHouse
curl http://localhost:8123/ping
```

```bash Redis
docker exec -it $(docker compose ps -q redis) redis-cli ping
```
</CodeGroup>
